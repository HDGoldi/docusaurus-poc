<task type="auto">
  <name>Task 1: Update CloudFormation template for dual-domain support</name>
  <files>infra/template.yaml</files>
  <action>
Modify infra/template.yaml with these changes:

1. Add a new parameter `AlternateDomainName` with default `help.1nce.com`:
   ```yaml
   AlternateDomainName:
     Type: String
     Default: help.1nce.com
     Description: Additional custom domain for the production site
   ```

2. Update the `Certificate` resource to add SubjectAlternativeNames covering both domains:
   ```yaml
   Certificate:
     Type: AWS::CertificateManager::Certificate
     Properties:
       DomainName: !Ref DomainName
       SubjectAlternativeNames:
         - !Ref DomainName
         - !Ref AlternateDomainName
       ValidationMethod: DNS
       DomainValidationOptions:
         - DomainName: !Ref DomainName
           HostedZoneId: !Ref HostedZoneId
       Tags:
         - Key: environment
           Value: dev
         - Key: component
           Value: ai
   ```

   IMPORTANT: Do NOT add a DomainValidationOptions entry for AlternateDomainName because its hosted zone is in a different AWS account. CloudFormation cannot auto-validate it. The user will manually add the DNS validation CNAME record in the external account.

3. Update the `ProdDistribution` Aliases to include both domains:
   ```yaml
   Aliases:
     - !Ref DomainName
     - !Ref AlternateDomainName
   ```

4. Leave all other resources unchanged (PreviewDistribution, buckets, IAM role, Route 53 records for help.1nce.ai).
  </action>
