- timestamp: 2026-03-21T00:00:00Z
  checked: sidebars/api.ts
  found: Uses `{type: 'autogenerated', dirName: '.'}` - delegates entirely to Docusaurus autogeneration which scans the filesystem
  implication: No explicit ordering is defined; Docusaurus will use filesystem/alphabetical ordering by default

  found: These are generated by docusaurus-plugin-openapi-docs and define internal structure per API spec, but they are NOT consumed by sidebars/api.ts which uses autogenerated mode
  implication: The per-spec sidebar.ts files are unused in the current autogenerated setup

- timestamp: 2026-03-21T00:00:00Z
  checked: docusaurus.config.ts OpenAPI plugin config
  found: No ordering/position config exists in the plugin config; it only defines specPath, outputDir, and sidebarOptions per spec
  implication: The plugin does not control top-level category ordering

## Resolution

root_cause: The API sidebar (`sidebars/api.ts`) uses `{type: 'autogenerated', dirName: '.'}` which delegates sidebar generation to Docusaurus's filesystem scanner. Docusaurus autogenerated sidebars sort items alphabetically by directory name when no `_category_.json` metadata files provide explicit `position` values. No `_category_.json` files exist in any of the six API subdirectories (`1nce-os`, `authorization`, `order-management`, `product-information`, `sim-management`, `support-management`), so they appear in ASCII/alphabetical order.
