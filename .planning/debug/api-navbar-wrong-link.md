## Resolution

root_cause: |
  The navbar item uses `type: 'docSidebar'` (line 184 of docusaurus.config.ts). This navbar type
  automatically resolves its link to the FIRST document found in the referenced sidebar. Since
  sidebars/api.ts uses `{type: 'autogenerated', dirName: '.'}`, the first document is determined
  by filesystem ordering: the `1nce-os/` directory comes first alphabetically, and within it
  `1-nce-os.info.mdx` has `sidebar_position: 0`, making it the first document. So the navbar
  resolves to `/api/1nce-os/1-nce-os`.

  The `docs/api/index.md` landing page (with `slug: /`) IS served at `/api`, but `type: 'docSidebar'`
  does not use it as the link target. It picks the first leaf document in the sidebar tree instead.
