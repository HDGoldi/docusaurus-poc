# Requirements: 1NCE Developer Hub — Client-Side Search

**Defined:** 2026-04-05
**Core Value:** Developers can browse all existing documentation and interactively test API endpoints exactly as they can on the current ReadMe.com-hosted site.

## v1.4 Requirements

Requirements for client-side search. Each maps to roadmap phases.

### Search Core

- [ ] **SRCH-01**: User can search across all 298 documentation pages via client-side full-text search
- [ ] **SRCH-02**: User can search across all 125 API endpoint pages generated by docusaurus-openapi-docs
- [ ] **SRCH-03**: User can search within page content (headings, paragraphs, code blocks)
- [ ] **SRCH-04**: User can filter search results by context tab (Documentation vs API)

### Search UI

- [ ] **UI-01**: Search bar renders in the main navbar on the right side, next to Terms and Abbreviations
- [ ] **UI-02**: Clicking the search bar opens an overlay with a slightly darkened background
- [ ] **UI-03**: Type-ahead suggestions appear as the user types, showing matching pages
- [ ] **UI-04**: User can click a search result to navigate directly to that page
- [ ] **UI-05**: Search overlay styled to match 1NCE branding (navy/teal theme, light-only mode)
- [ ] **UI-06**: User can open search via keyboard shortcut (Cmd/K or Ctrl/K)
- [ ] **UI-07**: Matching terms are highlighted in search results

### Deployment

- [ ] **DEPL-01**: Search index uses hashed filenames for CloudFront cache-busting
- [ ] **DEPL-02**: Smoke test script verifies search index URL returns 200 after deploy

## Future Requirements

### Search Enhancements

- **SRCH-05**: Stop-word removal tuning for IoT/telecom domain terms
- **SRCH-06**: Search analytics via PostHog (track queries, click-through)
- **SRCH-07**: Search result ranking tuning (boost titles over body content)

## Out of Scope

| Feature | Reason |
|---------|--------|
| Algolia DocSearch | Requires approval process, adds external dependency — deferred per PROJECT.md |
| Server-side / vector search | Adds cost and infrastructure — client-side sufficient for ~423 pages |
| Search in dev mode | Fundamental limitation of build-time index — plugin only works with `build + serve` |
| AI-powered "Ask AI" search | Separate feature (v1.1 AI Assistant) — not part of search milestone |
