Create `.github/workflows/rag-sync.yml` with the following exact structure (per D-01 through D-09):

**Workflow name:** `RAG Content Sync`

**Triggers (per D-02, D-04, D-05):**
- `push` to `branches: [main]` with `paths` filter:
  - `docs/**`
  - `specs/**`
  - `scripts/prepare-rag-content/**`
  - `scripts/prepare-rag-content.ts`
  - `scripts/sync-rag-to-s3.ts`
- `workflow_dispatch` (no inputs needed -- manual trigger for debugging/forced re-ingestion)

**Permissions:** `contents: read` (minimal permissions, read-only repo access)

**Concurrency (per D-03, mirrors gh-pages.yml pattern):**
```yaml
concurrency:
  group: rag-sync
  cancel-in-progress: true
```

**Single job `sync`** (Claude's discretion -- single job avoids artifact passing overhead since prepare outputs to `.rag-content/` which sync reads from the same filesystem):
- `runs-on: ubuntu-latest`

**Steps in order:**

1. `actions/checkout@v4`

2. `actions/setup-node@v4` with `node-version: 20` and `cache: npm` (matches deploy.yml pattern)

3. `run: npm ci` (plain npm ci, NO --omit=dev flag -- scripts need devDependencies like @aws-sdk/*)

4. Step named `Preprocess RAG content`:
   `run: npx tsx scripts/prepare-rag-content.ts`

5. `aws-actions/configure-aws-credentials@v6` with STATIC KEYS (per D-08, NOT OIDC):
   ```yaml
   aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
   aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
   aws-region: eu-central-1
   ```

6. Step named `Sync to S3 and start ingestion` with env block (per D-09) and run command:
   ```yaml
   env:
     RAG_CONTENT_BUCKET: ${{ vars.RAG_CONTENT_BUCKET }}
     KB_ID: ${{ vars.KB_ID }}
     DATA_SOURCE_ID: ${{ vars.DATA_SOURCE_ID }}
   run: npx tsx scripts/sync-rag-to-s3.ts --start-ingestion
   ```

No polling for ingestion completion (per D-07 -- fire-and-forget).

**CICD-02 compliance:** This is a NEW file. deploy.yml is NOT modified.
