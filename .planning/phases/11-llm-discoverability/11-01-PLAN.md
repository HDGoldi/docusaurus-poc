Create `plugins/llms-txt-plugin.ts` — a custom Docusaurus plugin that generates `llms.txt` during the `postBuild` lifecycle hook.

**Plugin structure:**

1. **Imports:** `@docusaurus/types` for `LoadContext` and `Plugin` types, `fs`, `path`, `gray-matter` (for frontmatter parsing).

2. **Constants — Section mapping per D-03:**
```typescript
const SECTION_MAP: Record<string, {product: 'connect' | 'os', label: string}> = {
  'introduction':          {product: 'connect', label: 'Introduction'},
  'connectivity-services': {product: 'connect', label: 'Connectivity Services'},
  'network-services':      {product: 'connect', label: 'Network Services'},
  'sim-cards':             {product: 'connect', label: 'SIM Cards'},
  'platform-services':     {product: 'connect', label: 'Platform Services'},
  'blueprints-examples':   {product: 'connect', label: 'Blueprints & Examples'},
  '1nce-portal':           {product: 'connect', label: '1NCE Portal'},
  'mcp-server':            {product: 'connect', label: 'MCP Server'},
  'troubleshooting':       {product: 'connect', label: 'Troubleshooting'},
  '1nce-os':               {product: 'os',      label: '1NCE OS'},
};
```

3. **Constants — API spec landing pages per D-05/D-11:**
```typescript
const API_SPECS = [
  { title: 'Authorization', route: '/api/authorization/authorization/' },
  { title: 'SIM Management', route: '/api/sim-management/sim-management/' },
  { title: 'Order Management', route: '/api/order-management/order-management/' },
  { title: 'Product Information', route: '/api/product-information/product-information/' },
  { title: 'Support Management', route: '/api/support-management/support-management/' },
  { title: '1NCE OS API', route: '/api/1nce-os/1-nce-os/' },
];
```

4. **`scanDocs(docsDir: string)` helper function:**
   - Uses `fs.readdirSync(docsDir, {recursive: true, encoding: 'utf-8'})` to find all `.md` and `.mdx` files
   - Parses each file with `gray-matter` to extract `title`, `slug`, `sidebar_position`
   - Returns array of `{title, slug, relativePath, sidebarPosition}` objects
   - Skips files that are `_category_.json` or don't have frontmatter title

5. **`filePathToRoute(relativePath: string, slug: string | undefined, baseUrl: string)` helper:**
   - If `slug` exists: return `${baseUrl}docs${slug.startsWith('/') ? slug : '/' + slug}`
   - Strip `docs/documentation/` prefix from relativePath for route derivation
   - Convert remaining path: remove file extension, replace `index` at end with empty string
   - Ensure trailing slash per D-07
   - Prepend `${baseUrl}docs/` to get full route path
   - Per D-07, construct absolute URL: `${siteUrl}${route}` where siteUrl is `siteConfig.url`

6. **`generateSectionLinks(docs, sectionDir, siteUrl, baseUrl)` helper:**
   - Filter docs where `relativePath` starts with `sectionDir + '/'`
   - Sort by `sidebarPosition` (ascending, undefined last)
   - Map to markdown links: `- [${title}](${absoluteUrl})`
   - Return joined string with newlines

7. **`postBuild({outDir, siteConfig})` main logic:**
   - Read template from `path.join(context.siteDir, 'static', 'llms-template.txt')`
   - Scan docs: `scanDocs(path.join(context.siteDir, 'docs', 'documentation'))`
   - Construct `siteUrl` from `siteConfig.url` (e.g., `https://help.1nce.com`)
   - Construct `baseUrl` from `siteConfig.baseUrl` (e.g., `/` or `/docusaurus-poc/`)
   - For each `<!-- LINKS:section-name -->` marker in template:
     - If `section-name` is `api`: generate API spec links from `API_SPECS` array, each as `- [${spec.title}](${siteUrl}${baseUrl}${spec.route})` (fix double slashes)
     - Otherwise: call `generateSectionLinks` for that section directory
   - Replace each marker with its generated link list
   - Write result to `path.join(outDir, 'llms.txt')`
   - Log: `[llms-txt-plugin] Generated llms.txt with N doc links and 6 API links`

8. **Export:** Default export function `llmsTxtPlugin(context: LoadContext): Plugin` returning `{name: 'llms-txt-plugin', postBuild: async function}`.

**Edge cases to handle:**
- `introduction/introduction-welcome.md` has `slug: /` — this maps to `${siteUrl}${baseUrl}docs/` (the docs root), per Pitfall 1
- `troubleshooting/` has only 1 file (`troubleshooting-technical-support.md`) — include it normally, it exists as a real page
- `index.md` files — if present in a section, include as the first link (they are category landing pages)
- URL construction must avoid double slashes when joining `siteUrl`, `baseUrl`, and route segments
- Handle both `baseUrl: '/'` (production) and `baseUrl: '/docusaurus-poc/'` (GitHub Pages) per Pitfall 4
