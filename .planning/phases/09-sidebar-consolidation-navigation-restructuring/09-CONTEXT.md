# Phase 9: Sidebar Consolidation & Navigation Restructuring - Context

**Gathered:** 2026-04-02
**Status:** Ready for planning

<domain>
## Phase Boundary

Merge 3 separate Docusaurus plugin instances (platform, blueprints, terms) into the main Documentation docs instance, reduce the navbar from 5 doc tabs to 2 (Documentation + API Explorer), enforce the original 11-section sidebar order, and redirect all old URLs to new locations. This phase covers NAV-02, NAV-03, NAV-04.

</domain>

<decisions>
## Implementation Decisions

### Content Migration Approach
- **D-01:** Physically move content files from `docs/platform/`, `docs/blueprints/`, and `docs/terms/` into `docs/documentation/` subdirectories
- **D-02:** Platform subdirs (1nce-os, 1nce-portal, platform-services) are moved directly as top-level sections in `docs/documentation/` — no intermediate `platform/` parent
- **D-03:** Keep current directory names as-is (no renaming during move)
- **D-04:** Remove the 3 plugin instances (`platform`, `blueprints`, `terms`) from `docusaurus.config.ts` and delete their sidebar files (`sidebars/platform.ts`, `sidebars/blueprints.ts`, `sidebars/terms.ts`)

### Sidebar Section Ordering
- **D-05:** Use `_category_.json` with `position` fields (1-11) to enforce the exact sidebar order: Introduction (1), 1NCE Portal (2), SIM Cards (3), MCP Server (4), Connectivity Services (5), Platform Services (6), Network Services (7), 1NCE OS (8), Troubleshooting (9), Blueprints & Examples (10), Terms & Abbreviations (11)
- **D-06:** Set explicit `label` values in `_category_.json` to match original ReadMe.com hub sidebar text exactly (e.g. "Blueprints & Examples" not "blueprints-examples")
- **D-07:** Keep `sidebars/documentation.ts` using autogenerated sidebar — position fields handle ordering

### Redirect Strategy
- **D-08:** Use `@docusaurus/plugin-client-redirects` for all old URL redirects — no CloudFront Function changes
- **D-09:** Generate redirect map programmatically from old path structure (`/platform/*`, `/blueprints/*`, `/terms/*`) to new `/docs/*` paths

### URL Structure
- **D-10:** Merged content uses flat `/docs/{section}/...` paths — e.g. `/docs/1nce-portal/dashboard`, `/docs/blueprints-examples/recipes`
- **D-11:** `terms-abbreviations.md` becomes a standalone doc at `/docs/terms-abbreviations` — not wrapped in a category directory

### Navbar Cleanup
- **D-12:** Remove 3 navbar items (1NCE Platform, Blueprints & Examples, Terms & Abbreviations) — keep only Documentation and API Explorer on the left, 3 external links on the right

### Claude's Discretion
- Exact redirect mapping for edge cases (index pages, nested paths)
- Whether to add `@docusaurus/plugin-client-redirects` as new dependency or if already installed
- Handling of any `_category_.json` files that already exist in moved directories

</decisions>
