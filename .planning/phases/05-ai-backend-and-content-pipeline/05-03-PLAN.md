**scripts/sync-rag-to-s3.ts** — Upload preprocessed chunks to S3 per CONTENT-03:

1. Read environment variable `RAG_CONTENT_BUCKET` (required, error if not set). Also accept `AWS_REGION` (default: `eu-central-1`).
2. Read `.rag-content/manifest.json` to confirm chunks exist. Error if manifest missing.
3. Glob `.rag-content/chunks/*` to get all .txt and .metadata.json files.
4. For each file, use `@aws-sdk/client-s3` `PutObjectCommand` to upload:
   - Key: `chunks/{filename}` (preserving the chunks/ prefix that the Bedrock data source expects)
   - Body: file content (read with fs.readFileSync)
   - ContentType: `text/plain` for .txt, `application/json` for .metadata.json
5. Use concurrent uploads (batch of 20 at a time using Promise.all with slicing) for performance.
6. After all uploads complete, print summary: `Uploaded {N} files to s3://{bucket}/chunks/`.
7. Optionally (if `--start-ingestion` flag passed): Call `StartIngestionJobCommand` from `@aws-sdk/client-bedrock-agent` to trigger Bedrock KB re-indexing. Requires `KB_ID` and `DATA_SOURCE_ID` env vars.
8. Install `@aws-sdk/client-s3` and `@aws-sdk/client-bedrock-agent` as devDependencies in project root (they are only used at build/deploy time, not at runtime in the Lambda).

Usage: `RAG_CONTENT_BUCKET=1nce-devhub-rag-content npx tsx scripts/sync-rag-to-s3.ts`
With ingestion: `RAG_CONTENT_BUCKET=... KB_ID=... DATA_SOURCE_ID=... npx tsx scripts/sync-rag-to-s3.ts --start-ingestion`

**scripts/deploy-chat-lambda.sh** — Build and deploy Lambda handler:

```bash
#!/usr/bin/env bash
set -euo pipefail

# Configuration
FUNCTION_NAME="${FUNCTION_NAME:-1nce-devhub-chat}"
REGION="${AWS_REGION:-eu-central-1}"
HANDLER_DIR="lambda/chat-handler"

echo "Building Lambda handler..."
cd "$HANDLER_DIR"
npm ci --production=false
npm run build
cd dist

echo "Creating deployment package..."
zip -r ../deployment.zip index.mjs

cd ../..

echo "Updating Lambda function code..."
aws lambda update-function-code \
  --function-name "$FUNCTION_NAME" \
  --zip-file "fileb://${HANDLER_DIR}/deployment.zip" \
  --region "$REGION"

echo "Waiting for update to complete..."
aws lambda wait function-updated \
  --function-name "$FUNCTION_NAME" \
  --region "$REGION"

echo "Lambda function updated successfully."
echo "Function URL: $(aws lambda get-function-url-config --function-name "$FUNCTION_NAME" --region "$REGION" --query 'FunctionUrl' --output text)"
```

Make the script executable: `chmod +x scripts/deploy-chat-lambda.sh`
