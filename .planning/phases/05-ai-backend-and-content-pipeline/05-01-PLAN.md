**types.ts** — Shared types per D-01, D-02, D-03:
```typescript
export interface Chunk {
  id: string;           // e.g. "docs-introduction-welcome-001"
  heading: string;      // Section heading text
  headingLevel: number; // 2 or 3 (h2/h3)
  content: string;      // Plain markdown text
  startLine: number;
}

export interface ChunkMetadata {
  url: string;          // e.g. "/docs/introduction-welcome/"
  title: string;        // Page title from frontmatter
  breadcrumb: string;   // e.g. "Documentation > Introduction > Welcome"
  content_type: string; // "guide" | "tutorial" | "api-reference" | "glossary"
  category: string;     // Top-level directory name
  section_heading: string; // The h2/h3 heading for this chunk
}

export interface EndpointChunk {
  id: string;           // e.g. "api-authorization-obtain-access-token"
  method: string;       // "GET", "POST", etc.
  path: string;         // "/api/v1/authenticate"
  summary: string;
  description: string;
  parameters: string;   // Formatted parameter list
  responses: string;    // Formatted response list
  tags: string[];
}
```

**strip-mdx.ts** — MDX stripping using unified/remark per research code example:
- Import unified, remark-parse, remark-mdx, remark-stringify, unist-util-visit, gray-matter
- Export `stripMdx(content: string): Promise<{ frontmatter: Record<string, any>; plainMarkdown: string }>`
- Remove node types: mdxjsEsm, mdxJsxFlowElement, mdxJsxTextElement, mdxFlowExpression
- Use the exact implementation from 05-RESEARCH.md "MDX Stripping with Unified/Remark" code example
- Additionally remove any `import` lines that survive as raw text (some MDX files have plain import statements)

**chunk-by-headings.ts** — Split by h2/h3 boundaries per D-01:
- Export `chunkByHeadings(markdown: string, docId: string): Chunk[]`
- Split on lines matching `^(#{2,3})\s+(.+)` regex
- Include heading line in chunk content
- If no h2/h3 found, treat entire document as single chunk with id `${docId}-001`
- Use the exact implementation from 05-RESEARCH.md "Chunk by Heading Boundaries" code example

**extract-openapi.ts** — OpenAPI endpoint extraction per D-02:
- Export `extractEndpoints(specPath: string, specName: string): EndpointChunk[]`
- Read JSON spec, iterate over `spec.paths` entries
- For each method (get/post/put/patch/delete), extract operationId, summary, description, parameters, responses
- Format parameters as `- name (in, required/optional): description`
- Format responses as `- statusCode: description`
- Use the exact implementation from 05-RESEARCH.md "OpenAPI Endpoint Extraction" code example

**generate-metadata.ts** — Metadata sidecar generation per D-03:
- Export `generateMetadata(opts: { url: string; title: string; breadcrumb: string; contentType: string; category: string; sectionHeading: string }): { metadataAttributes: ChunkMetadata }`
- Return object in Bedrock metadata sidecar format: `{ metadataAttributes: { url, title, breadcrumb, content_type, category, section_heading } }`
- Export `deriveBreadcrumb(filePath: string, title: string): string` — convert file path like `docs/documentation/connectivity-services/data-services.mdx` to `Documentation > Connectivity Services > Data Services`
- Export `deriveContentType(filePath: string): string` — return "api-reference" for paths containing `/api/`, "glossary" for paths containing `/glossary/`, "guide" for everything else
- Export `deriveUrl(filePath: string): string` — convert `docs/documentation/connectivity-services/data-services.mdx` to `/docs/connectivity-services/data-services/`

Install required dependencies: `npm install unified remark-parse remark-mdx remark-stringify unist-util-visit`

**scripts/prepare-rag-content.ts** — Main orchestrator per D-04:

1. **Discover docs files:** Use glob to find all `docs/**/*.{md,mdx}` files. Skip any files in `docs/api/` since those are generated MDX from OpenAPI plugin (per Research Pitfall 6 — they are mostly JSX/base64, not useful text).

2. **Process each doc file:**
   - Read file content
   - Call `stripMdx(content)` to get frontmatter + plain markdown
   - Call `chunkByHeadings(plainMarkdown, docId)` where docId is derived from file path (e.g., `docs-documentation-connectivity-services-data-services`)
   - For each chunk, generate metadata using `generateMetadata()` with:
     - url: `deriveUrl(filePath)`
     - title: frontmatter.title or filename
     - breadcrumb: `deriveBreadcrumb(filePath, title)`
     - contentType: `deriveContentType(filePath)`
     - category: first directory under docs/ (e.g., "documentation", "platform", "blueprints")
     - sectionHeading: chunk.heading
   - Write chunk content to `.rag-content/chunks/{chunkId}.txt`
   - Write metadata to `.rag-content/chunks/{chunkId}.txt.metadata.json`

3. **Process OpenAPI specs:** Glob `specs/*.json` (6 files: 1nce-os.json, authorization.json, order-management.json, product-information.json, sim-management.json, support-management.json).
   - For each spec, call `extractEndpoints(specPath, specName)`
   - For each endpoint chunk, format content as plain text:
     ```
     # {METHOD} {path}

     {summary}

     {description}

     ## Parameters
     {parameters}

     ## Responses
     {responses}
     ```
   - Generate metadata with content_type "api-reference", category from spec name, url derived from spec + operationId
   - Write to `.rag-content/chunks/{endpointChunkId}.txt` and `.rag-content/chunks/{endpointChunkId}.txt.metadata.json`

4. **Write manifest:** Create `.rag-content/manifest.json` with:
   ```json
   {
     "generated": "ISO timestamp",
     "total_chunks": N,
     "doc_chunks": N,
     "api_chunks": N,
     "docs_processed": N,
     "specs_processed": N,
     "skipped_api_mdx": N
   }
   ```

5. **Console output:** Print summary stats (total chunks, docs processed, specs processed, output directory).

6. **Clean before run:** Delete `.rag-content/` directory at start to ensure fresh output.
