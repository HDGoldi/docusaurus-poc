Step 1: Build the site and measure index size.
```bash
cd /Users/Jan.Sulaiman/_Projects/docusaurus_poc
npm run build
# Measure search index size (D-09: optimize only if > 5MB compressed)
ls -la build/search-index-*.json 2>/dev/null || ls -la build/search-index*.json 2>/dev/null
# Check gzipped size
gzip -c build/search-index-*.json 2>/dev/null | wc -c
```

Expected: Index ~4.0MB uncompressed, ~844KB gzipped (from Phase 13 measurement). If under 5MB, D-09 confirms NO optimization needed.

Step 2: Start the serve process and inspect rendered DOM.
```bash
npx docusaurus serve --port 3456 &
# Wait for server, then use curl or browser tools to verify
```

Step 3: Inspect the search overlay DOM structure to verify CSS selectors.
Use the `/browse` skill or manual browser inspection at http://localhost:3456 to:
1. Open search (click search bar or Cmd+K)
2. Type a query (e.g., "SIM card")
3. Check the actual DOM class names on:
   - The dropdown container (is it `[class*="dropdownMenu"]`?)
   - The result items (is it `[class*="hitWrapper"]`?)
   - The mark elements inside results (is it `[class*="hitWrapper"] mark`?)
   - Whether a backdrop element exists or if the ::before approach works

Step 4: If CSS selectors don't match actual DOM:
- Update src/css/custom.css with corrected selectors
- The ::before pseudo-element backdrop is the most likely item to need adjustment
- Common fixes:
  - If no backdrop visible: Try targeting `body` when search modal is open (the plugin may add a class or attribute to body)
  - If mark styling doesn't apply: Increase specificity or check if plugin uses a different wrapper class
  - If CSS variables don't take effect: Check if plugin reads them at build time vs runtime

Step 5: Test D-10 (title boosting) empirically:
- Search for a term that appears in both a page title and page body (e.g., "SIM card")
- Check if the page whose TITLE contains "SIM card" appears above pages where it only appears in body text
- This is observational only — if titles already rank first, D-10 is satisfied by default plugin behavior
- If titles don't rank first, note in the summary that D-10 cannot be achieved without custom lunr config (per research finding)

Step 6: Kill the serve process after inspection.
