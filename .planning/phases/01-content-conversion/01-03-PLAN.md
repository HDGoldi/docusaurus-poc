       **Input:** `docs/` directory
       **Output:** Same files with HTMLBlocks converted

       The script must handle four HTMLBlock categories:

       **Category A -- Centered image wrappers (~30 instances):**
       - Pattern: `<HTMLBlock>{` followed by HTML containing `<img src="data:image/..." />` or `<img src="https://..." />`
       - Action: Strip `<HTMLBlock>{`` `` and `` `` `}</HTMLBlock>` wrapper, the `<img>` tag inside should already have been converted to Markdown by plan 02's base64/image scripts. If an `<img>` tag remains, convert it to `![alt](src)`.

       **Category B -- CSS grid/table layouts (~5-8 instances, including welcome page):**
       - Pattern: HTMLBlock containing `<div style="display: grid">` or `<div class="navigationTable">` or similar CSS layout constructs
       - Action: Create `src/components/NavigationGrid.tsx` as a React component that accepts `items` prop (array of `{title, description, link}`) and renders a responsive CSS grid. Replace the HTMLBlock with an import + `<NavigationGrid items={[...]} />` call.
       - The NavigationGrid component should:
         ```tsx
         import React from 'react';
         import styles from './NavigationGrid.module.css';

         interface NavItem {
           title: string;
           description: string;
           link: string;
           icon?: string;
         }

         interface Props {
           items: NavItem[];
         }

         export default function NavigationGrid({ items }: Props): JSX.Element {
           return (
             <div className={styles.grid}>
               {items.map((item, i) => (
                 <a key={i} href={item.link} className={styles.card}>
                   {item.icon && <span className={styles.icon}>{item.icon}</span>}
                   <h3>{item.title}</h3>
                   <p>{item.description}</p>
                 </a>
               ))}
             </div>
           );
         }
         ```
       - Also create `src/components/NavigationGrid.module.css` with grid layout styles

       **Category C -- Simple HTML wrappers (~15 instances):**
       - Pattern: `<HTMLBlock>{` followed by simple `<div>`, `<span>`, `<p>` content
       - Action: Strip `<HTMLBlock>{`` `` and `` `` `}</HTMLBlock>` wrapper, then:
         - Replace `class=` with `className=`
         - Self-close void elements: `<img ...>` -> `<img ... />`, `<br>` -> `<br />`, `<hr>` -> `<hr />`
         - Escape any unescaped `{` or `}` that are NOT part of JSX expressions (MDX treats `{` as JSX)

       **Category D -- Styled content with `<style>` tags (~5 instances):**
       - Pattern: HTMLBlock containing `<style>...</style>` followed by HTML using those styles
       - Action: Extract the CSS into the component's CSS module or inline styles. Convert `<style>` block content to inline `style={{...}}` JSX attributes on the elements. Strip the HTMLBlock wrapper.

       **General approach for all categories:**
       - The wrapper extraction regex must handle nested backticks (pitfall 5 from research):
         Find `<HTMLBlock>{\`` and scan forward to matching `\`}</HTMLBlock>`, not just the first backtick
       - After stripping the wrapper, classify the inner content into one of the 4 categories
       - Log each HTMLBlock with its category and file path for debugging

       **Input:** `docs/` directory
       **Output:** Same files with `<Table>` JSX converted to GFM Markdown tables

       a. Find all .md files in `docs/`
       b. For each file, find `<Table ... >` to `</Table>` blocks (may include `align={[...]}` attribute)
       c. Extract the HTML table structure inside:

          - `<tbody>` -> data rows
          - `<tr>` -> rows
          - `<th>` or `<td>` -> cells

          ```
          | Header 1 | Header 2 |
          | --- | --- |
          | Cell 1 | Cell 2 |
          ```
       e. Handle edge cases:
          - Cells containing Markdown links or inline code
          - Cells with newlines (replace with spaces)
          - Tables with no `<thead>` (use first row as header)
       f. Replace the entire `<Table>...</Table>` block with the Markdown table
       g. Log: total tables found, converted

    1. Create `scripts/08-convert-links.ts` (per CONV-06, research Pattern 2 and Example 2):

       **Input:** `docs/` directory
       **Output:** Same files with `(doc:slug)` links converted to Docusaurus internal links

       The script must:
       a. Build the slug map using `buildSlugMap('docs/')` from utils/slug-map.ts
       b. Log the slug map size (expected: ~250+ entries)
       c. Find all .md files in `docs/`
       d. For each file, find all `(doc:slug)` patterns using regex:
          `\[([^\]]*)\]\(doc:([a-z0-9-]+)(#[a-z0-9-]*)?\)`
          This matches: `[link text](doc:some-slug)` and `[link text](doc:some-slug#anchor)`
       e. For each match:
          - Look up the slug in the slug map
          - If found: replace with `[link text](/resolved/path)` or `[link text](/resolved/path#anchor)`
          - If NOT found: log a warning with the slug and source file, leave the link text but convert to a commented-out placeholder: `[link text](/unresolved/doc:slug)` so it is still visible but clearly marked
       f. Also handle bare `(doc:slug)` without markdown link syntax (some may appear as plain text references)
       g. Log: total links found, resolved, unresolved (with list of unresolved slugs)
       h. For links that appear to target API reference content (slugs matching API endpoint names), convert to `/api/{slug}` placeholder path for Phase 2

    2. Create `scripts/09-convert-admonitions.ts` (per CONV-07, research Example 3):

       **Input:** `docs/` directory
       **Output:** Same files with blockquote admonitions converted to Docusaurus `:::` syntax

       The script must:
       a. Find all .md files in `docs/`
       b. For each file, find blockquote admonition patterns:
          `^> :(\w+):\s*(.+(?:\n>.*)*)`
          This matches multi-line blockquote admonitions starting with `> :warning:`, `> :info:`, etc.
       c. Map ReadMe emoji types to Docusaurus admonition types:
          - `:warning:` -> `:::warning`
          - `:info:` -> `:::info`
          - `:memo:` -> `:::note`
          - `:bulb:` -> `:::tip`
          - `:check:` -> `:::note`
          - `:x:` -> `:::danger`
          - Any unknown type -> `:::note` (with a log warning)
       d. Convert the blockquote content:
          - Strip leading `> ` from each line
          - Wrap in `:::type\n{content}\n:::`
       e. Log: total admonitions found, converted, by type
