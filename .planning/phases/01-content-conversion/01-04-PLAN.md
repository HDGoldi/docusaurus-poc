    1. Create `scripts/run-pipeline.ts` (per D-02, D-04, CONV-10, research Pattern 1):

       **What it does:** Runs all 9 conversion steps in order with timing and error reporting.

       ```typescript
       import { execSync } from 'child_process';
       import { log, error, success } from './utils/logger';

       const STEP_NAME = 'pipeline';

       const steps = [
         { script: '01-copy-and-normalize', desc: 'Copy and normalize folder names' },
         { script: '02-convert-frontmatter', desc: 'Convert frontmatter' },
         { script: '03-extract-base64', desc: 'Extract base64 images' },
         { script: '04-download-images', desc: 'Download remote images' },
         { script: '05-convert-images', desc: 'Convert Image JSX tags' },
         { script: '06-convert-htmlblocks', desc: 'Convert HTMLBlocks' },
         { script: '07-convert-tables', desc: 'Convert Table JSX' },
         { script: '08-convert-links', desc: 'Convert doc:slug links' },
         { script: '09-convert-admonitions', desc: 'Convert admonitions' },
       ];

       let failed = 0;
       const startTime = Date.now();

       for (const step of steps) {
         const stepStart = Date.now();
         log(STEP_NAME, `Running: ${step.desc}...`);
         try {
           execSync(`npx tsx scripts/${step.script}.ts`, {
             stdio: 'inherit',
             cwd: process.cwd(),
           });
           const elapsed = ((Date.now() - stepStart) / 1000).toFixed(1);
           success(STEP_NAME, `${step.desc} completed (${elapsed}s)`);
         } catch (e) {
           error(STEP_NAME, `${step.desc} FAILED`);
           failed++;
           // Continue to next step -- allow partial pipeline runs
         }
       }

       const totalElapsed = ((Date.now() - startTime) / 1000).toFixed(1);
       if (failed > 0) {
         error(STEP_NAME, `Pipeline completed with ${failed} failures in ${totalElapsed}s`);
         process.exit(1);
       } else {
         success(STEP_NAME, `Pipeline completed successfully in ${totalElapsed}s`);
       }
       ```


    3. Run `npx docusaurus build` and capture ALL errors:
       ```bash
       npx docusaurus build 2>&1
       ```

    4. **Fix-and-rebuild loop:** For each build error:
       a. Identify the failing file and error type (MDX parse error, broken link, missing import, etc.)
       b. Determine if the fix should be in:
          - A conversion script (pattern not fully handled) -> fix script, re-run pipeline
          - The specific .md file (one-off edge case) -> fix directly BUT also add the fix to the relevant conversion script for idempotency
          - docusaurus.config.ts (configuration issue) -> fix config
          - sidebars.ts (missing/wrong entry) -> fix sidebar generator
       c. Apply the fix
       d. Re-run `npx docusaurus build`
       e. Repeat until build succeeds with zero errors

    5. Common build errors to expect and fix:
       - **"Could not parse expression"**: Unescaped `{` or `}` in MDX content -> escape as `\{` and `\}`
       - **"Unexpected token"**: Remaining JSX-like syntax that MDX parser chokes on -> convert to HTML entities or remove
       - **"Broken link"**: `onBrokenLinks: 'throw'` catches unresolved links -> fix link targets or temporarily set to 'warn' for Phase 1 (since API links resolve in Phase 2)
       - **"Unknown sidebar item"**: sidebar references a doc ID that does not exist -> fix sidebars.ts or the filename mapping
       - **`class` instead of `className`**: Remaining HTML with `class=` attribute -> convert to `className=`
       - **Unclosed tags**: `<img>`, `<br>`, `<hr>` without self-close -> add ` />`

    6. **Important:** For broken links pointing to API reference content (Phase 2), temporarily change `onBrokenLinks` to `'warn'` in docusaurus.config.ts if needed. Document which links are Phase 2 dependencies.

    7. After clean build, record final stats:
       - Total pages in build output
       - Total images in static/img/
       - Any warnings from the build
       - List of unresolved links (Phase 2 dependencies)
