**Step 3: Create `src/components/ChatWidget/useChatStream.ts`**

Custom hook encapsulating the POST-based SSE streaming logic. This is the core data-fetching mechanism (per D-07, CHAT-02).

The hook must:
- Accept `endpoint: string` and `question: string`
- Use `fetch` with `method: 'POST'`, `Content-Type: application/json`, body `{ question }`
- Read from `response.body.getReader()` with `TextDecoder`
- Buffer incoming chunks, split on `\n\n` to parse SSE events
- Handle `data: [DONE]` as stream termination
- Parse each `data: {...}` line as JSON `SSEEvent`
- Call `onText(content)` for `type: 'text'` events (appending to message)
- Call `onSources(sources)` for `type: 'sources'` events
- Call `onError(errorMessage)` for `type: 'error'` events — map known error patterns: if response status is 429 return `COPY.errorRateLimit`, if 503 return `COPY.errorBackend`, if network error (TypeError from fetch) return `COPY.errorNetwork`, else `COPY.errorGeneric`
- Use `AbortController` — expose `abort()` for the Stop button (per D-08)
- Return `{ sendMessage, abort, state }` where state is `ChatState`
- On `abort()`, do NOT treat as error — keep partial content, set state to `idle`
- On component unmount, call `abort()` via useEffect cleanup to prevent memory leaks (Pitfall 5 from research)

Key implementation detail for SSE buffer parsing (Pitfall 2 from research):

}
