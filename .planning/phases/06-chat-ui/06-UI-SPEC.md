# Phase 6 — UI Design Contract

> Visual and interaction contract for the Chat UI phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Docusaurus Infima) |
| Preset | not applicable |
| Component library | none — pure React components with CSS Modules |
| Icon library | none — single SVG chat bubble icon inline |
| Font | Barlow, sans-serif (already loaded site-wide via `--ifm-font-family-base`) |

---

## Spacing Scale

Declared values (multiples of 4, aligned with Infima defaults):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gap in citation links, superscript marker offset |
| sm | 8px | Chat bubble internal padding (horizontal), gap between suggestion chips |
| md | 16px | Chat bubble internal padding (vertical), input field padding, gap between consecutive messages, drawer body padding |
| lg | 24px | Drawer header/footer padding, welcome message spacing |
| xl | 32px | Floating button offset from viewport edge (bottom + right) |

Exceptions:
- Floating button: 48px diameter (touch target, meets 44px minimum)
- Drawer width on desktop: 400px fixed
- Mobile breakpoint: full viewport width at `<=768px`

---

## Typography

All typography inherits Barlow from the site. Sizes are specific to the chat widget. Two weights only: 400 (regular) for all body text and labels, 600 (semibold) for the drawer title.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 15px | 400 (regular) | 1.5 | Chat message text, AI response prose |
| Label | 13px | 400 (regular) | 1.4 | Timestamp, "Sources" heading, "1NCE Docs Assistant" label, suggestion chip text |
| Input | 15px | 400 (regular) | 1.5 | Chat input field text |
| Drawer Title | 16px | 600 (semibold) | 1.3 | Drawer header: "1NCE Docs Assistant" |

- Size: 48px diameter circle
- Background: `var(--ifm-color-primary)` (#29abe2)
- Icon: white chat bubble SVG, 24px
- Z-index: `calc(var(--ifm-z-index-fixed) - 1)` (199, same layer as back-to-top)

- Close animation: slide out to right, 150ms ease-in
- Layout: flex column — header (fixed) | messages (scrollable) | input (fixed)

### ChatDrawer Header
- Height: 56px
- Padding: 0 24px
- Content: "1NCE Docs Assistant" (16px/600) left-aligned, close button (X icon, 20px) right-aligned

- Close button hover: `var(--ifm-color-primary)`

### ChatMessage (User Bubble)

- Background: `var(--ifm-color-primary)` (#29abe2)
- Text color: `#ffffff`

- Padding: 16px 16px

### ChatMessage (AI Bubble)
- Alignment: left (flex-start)
- Background: light `#f5f6f7` / dark `#2a2a2d`
- Text color: `var(--ifm-font-color-base)`
- Border-radius: 16px 16px 16px 4px (tail bottom-left)
- Padding: 16px 16px
- Max-width: 85% of message area
- Margin-bottom: 16px
- Font: 15px/1.5 regular

- Streaming cursor: blinking `|` character appended during streaming, `animation: blink 1s step-end infinite`

### CitationList (Sources Section)
- Position: below AI bubble content, inside the same bubble
- Separator: `1px solid var(--ifm-toc-border-color)`, 8px margin top/bottom
- Heading: "Sources" at 13px/400
- Each source: page title as link, 13px/400
- Link color: `var(--ifm-color-primary)`
- Link hover: underline
- Superscript markers in message text: `[1]` rendered as `<sup>` at 11px, color `var(--ifm-color-primary)`, cursor pointer

### SuggestionChips (Starter Questions)
- Layout: flex wrap, 8px gap
- Shown: only when message list is empty (welcome state)
- Each chip: pill shape, border `1px solid var(--ifm-color-primary)`, border-radius 20px
- Chip padding: 8px 16px
- Chip text: 13px/400, color `var(--ifm-color-primary)`
- Chip hover: background `var(--ifm-color-primary)`, text `#ffffff`
- Transition: `all 150ms ease`

### ChatInput (Input Area)
- Position: bottom of drawer, sticky
- Background: `var(--ifm-background-surface-color, #f5f6f7)` / dark `#2a2a2d`
- Border-top: `1px solid var(--ifm-toc-border-color)`
- Padding: 16px 24px
- Input field: full width minus button, 15px font, border-radius 24px, border `1px solid var(--ifm-toc-border-color)`
- Input padding: 16px 48px 16px 16px (right padding for button)
- Placeholder: "Ask a question..." at `var(--ifm-font-color-secondary, #999)`
- Submit: Enter key (single line input, no multiline)
- Send button: icon only (arrow-up), 32px circle, positioned inside input right edge, `aria-label="Send message"`
- Send button enabled: `var(--ifm-color-primary)` background, white icon
- Send button disabled (empty input): `#ccc` background (light) / `#555` background (dark)
- During streaming: send button becomes Stop button (square icon), `var(--ifm-color-primary)` background

### Loading Indicator
- Three dots bouncing animation inside an AI bubble placeholder
- Dot size: 8px circles
- Dot color: `var(--ifm-font-color-secondary, #999)`
- Animation: sequential bounce, 600ms total cycle
- Shown: after user sends message, before first SSE token arrives

### Error State (inline)
- Rendered as a system message (left-aligned, no bubble background)
- Icon: warning triangle, 16px, `#e3342f`
- Text color: `#e3342f`
- Font: 13px/1.4 regular

## Interaction Contract

### Drawer Open/Close
- Click floating button: drawer slides in from right (200ms ease-out), button hides
- Click close (X) button: drawer slides out to right (150ms ease-in), button reappears
- Press Escape key: same as close button
- Click outside drawer: no action (user may be reading content behind)

### Message Flow
1. User types in input, presses Enter or clicks send button
2. User message appears in chat as right-aligned bubble
3. Input clears, send button becomes Stop button, input disabled
4. Loading indicator (bouncing dots) appears as AI bubble placeholder
5. First SSE token arrives: loading indicator replaced with AI bubble, tokens append with blinking cursor
6. Tokens continue appending in real-time
7. `sources` event arrives: citation superscripts retroactively rendered in text, Sources section appended below message
8. `[DONE]` event: blinking cursor removed, input re-enabled, Stop button becomes send button
9. Auto-scroll to bottom on each new token (smooth)

### Stop Button
- Visible only during active streaming
- Click: calls `AbortController.abort()`, stops stream immediately
- AI message retains whatever tokens were received so far
- Input re-enables, send button returns

### Citation Click
- Click superscript `[N]` or source list link: drawer closes, browser navigates to cited page URL
- URLs are root-relative; prepend `siteConfig.baseUrl` if non-root (GitHub Pages)

### Navigation Reset
- On route change (pathname changes): all messages cleared, drawer closes if open, returns to welcome state
- On same-page drawer close/reopen: messages preserved

### Mobile Behavior (<=768px)
- Drawer becomes full-screen overlay (100vw x 100vh)
- Body scroll locked (`overflow: hidden` on `<html>`)
- No left border or shadow (edge-to-edge)
- Close button more prominent (same position, same size)

## Accessibility

| Concern | Implementation |
|---------|---------------|
| Keyboard navigation | Escape closes drawer; Tab cycles through interactive elements in drawer; focus trapped when drawer open |
| Focus management | On drawer open: focus moves to input field. On drawer close: focus returns to floating button. |
| Screen reader | Drawer has `role="dialog"`, `aria-label="Chat with 1NCE Docs Assistant"`. Floating button has `aria-label="Open AI chat assistant"`. Send button has `aria-label="Send message"`. Stop button has `aria-label="Stop generating"`. Close button has `aria-label="Close chat"`. |
| Live region | AI message area has `aria-live="polite"` for streaming content updates |
| Reduced motion | `@media (prefers-reduced-motion: reduce)`: disable slide animation (instant show/hide), disable bouncing dots (static dots), disable blinking cursor (static cursor) |
| Color contrast | User bubble: white on #29abe2 = 3.1:1 (AA Large). AI bubble text: #4a4a4a on #f5f6f7 = 5.4:1 (AA). Error text: #e3342f meets AA on both backgrounds. |
