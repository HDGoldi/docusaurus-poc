  <action>
1. Install the search plugin:
   ```bash
   npm install @easyops-cn/docusaurus-search-local@0.55.1
   ```

2. Edit `docusaurus.config.ts` — replace the `themes` array (currently line 313) from:
   ```typescript
   themes: ['docusaurus-theme-openapi-docs'],
   ```
   to:
   ```typescript
   themes: [
     'docusaurus-theme-openapi-docs',
     ['@easyops-cn/docusaurus-search-local', {
       // Core indexing
       indexDocs: true,
       indexBlog: false,
       indexPages: false,

       // Multi-instance: CRITICAL — both docs plugin instances
       docsRouteBasePath: ['/docs', '/api'],
       docsDir: ['docs/documentation', 'docs/api'],

       // Language
       language: ['en'],

       // Caching — content-hashed filenames for CloudFront
       hashed: 'filename',

       // UI behavior — plugin defaults per D-10
       searchBarShortcutHint: true,
       searchBarPosition: 'right',
       searchResultLimits: 8,
       explicitSearchResultPath: true,
     }],
   ],
   ```

   Key points:
   - Add as SECOND entry in `themes` array (after `docusaurus-theme-openapi-docs`). Per D-10, use plugin defaults — no CSS customization.
   - `docsRouteBasePath: ['/docs', '/api']` is CRITICAL — without the array, 125 API pages are silently excluded from search.
   - `docsDir: ['docs/documentation', 'docs/api']` must match the `path` values from the two docs plugin instances in the config.
   - `indexBlog: false` because blog is disabled in preset-classic.
   - `indexPages: false` because custom pages are not documentation content.
   - `hashed: 'filename'` produces content-hashed filenames compatible with CloudFront cache policy.
   - Do NOT add `ignoreFiles` yet — redirect stub exclusion happens in Plan 02 after inspecting build output.

3. Run the build to test Rspack compatibility:
   ```bash
   npm run build
   ```

4. **Rspack go/no-go gate (per D-01, D-02, D-03):**
   - If the build SUCCEEDS: check that `build/search-index-*.json` files exist. Grep for a known API title:
     ```bash
     grep -l "Obtain Access Token" build/search-index-*.json
     ```
     If API titles appear in the index, Rspack is compatible. Proceed without changes.
   - If the build FAILS or search index is missing or API pages are not in the index:
     Disable Rspack immediately (per D-01). Edit `docusaurus.config.ts` lines 18-21:
     ```typescript
     future: {
       v4: true,
       experimental_faster: false,  // Disabled to unblock search (D-01)
     },
     ```
     Then rebuild: `npm run build`
     Confirm search index now exists with API pages included.
     Do NOT spend time debugging the Rspack interaction (per D-01).

5. Record the Rspack compatibility outcome and index file size in the SUMMARY for Phase 14 reference:
   ```bash
   ls -lh build/search-index-*.json
   ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client browser -> search index JSON | Search index is a static JSON file served from S3/CloudFront. No server-side processing. |
| User input -> lunr.js client-side | Search queries are processed entirely in the browser by lunr.js. No data sent to any server. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Information Disclosure | search-index-*.json | accept | Search index contains only content already publicly visible on the site. No secrets, credentials, or draft content in the index. All indexed pages are already public HTML. |
| T-13-02 | Denial of Service | lunr.js client-side search | accept | Oversized index could freeze mobile browsers. Mitigated by Phase 14 optimization if index exceeds 5 MB (D-08). For Phase 13, accept current size per D-09. |
| T-13-03 | Tampering | search-index-*.json on CDN | accept | Index served over HTTPS from CloudFront. Content integrity protected by TLS. No additional signing needed for a public documentation index. |
</threat_model>
