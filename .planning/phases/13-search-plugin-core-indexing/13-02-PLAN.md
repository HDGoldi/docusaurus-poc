  <action>
1. Build the site first to generate redirect stub HTML files:
   ```bash
   npm run build
   ```

2. Identify all redirect stub HTML files in `build/`. Redirect stubs are small HTML files (typically <2KB) containing only a meta-refresh redirect. Find them:
   ```bash
   # List HTML files smaller than 2KB that contain meta-refresh (redirect stubs)
   find build -name "index.html" -size -2k -exec grep -l "http-equiv.*refresh" {} \;
   ```

3. Categorize the redirect stub paths to determine `ignoreFiles` patterns. Based on the redirect config in docusaurus.config.ts, the stubs will be generated at these path prefixes:
   - `/dev-hub/` — old ReadMe paths (lines 158-178)
   - `/blueprints-examples/` — old blueprint leaf pages (lines 182-215)
   - `/starting-guide/` — old starting-guide path (line 167)
   - `/platform/` — old platform prefix from createRedirects (line 244)
   - `/blueprints/` — old blueprints prefix from createRedirects (line 248)
   - `/1nce-os/`, `/network-services/`, `/platform-services/`, `/1nce-portal/`, `/connectivity-services/`, `/sim-cards/` — section-level redirects from createRedirects without `/docs/` prefix (lines 236-239)
   - `/terms/` — terms redirect from createRedirects (line 258)
   - `/examples-*` — root shortcut paths (lines 223-228)
   - `/recipes` — root shortcut (line 229)
   - Old `/index` suffix variants from createRedirects (lines 283-290)

4. After inspecting the actual `build/` output, add `ignoreFiles` to the search plugin config in `docusaurus.config.ts`. The `ignoreFiles` option accepts an array of regex patterns that match against the URL path of each page. Add to the search plugin options object (inside the existing `['@easyops-cn/docusaurus-search-local', { ... }]` entry):

   ```typescript
   ignoreFiles: [
     // Redirect stubs from plugin-client-redirects (D-07)
     // Old ReadMe /dev-hub/ paths
     /\/dev-hub\//,
     // Old /starting-guide/ path
     /\/starting-guide\//,
     // Old /platform/ prefix (not /docs/platform-services/)
     /^\/platform\//,
     // Old /blueprints/ and /blueprints-examples/ top-level (not /docs/blueprints-examples/)
     /^\/blueprints\//,
     /^\/blueprints-examples\//,
     // Old /terms/ prefix (not /docs/terms-abbreviations)
     /^\/terms\//,
     // Root shortcut redirects
     /^\/examples-/,
     /^\/recipes\//,
     // Section-level redirects without /docs/ prefix
     /^\/1nce-os\//,
     /^\/network-services\//,
     /^\/platform-services\//,
     /^\/1nce-portal\//,
     /^\/connectivity-services\//,
     /^\/sim-cards\//,
   ],
   ```

   **IMPORTANT:** The exact patterns MUST be verified against actual `build/` output from step 2. If the stubs are at different paths than expected, adjust the patterns accordingly. The goal is to exclude ALL redirect stubs (files containing meta-refresh) while keeping ALL real documentation and API pages.

5. Rebuild and verify redirect stubs are excluded:
   ```bash
   npm run build
   ```
   Then verify:
   ```bash
   # Count redirect stubs that should NOT be in index
   find build -name "index.html" -size -2k -exec grep -l "http-equiv.*refresh" {} \; | wc -l
   # Search for old ReadMe slugs in the index (should return 0)
   grep -c "introduction-welcome" build/search-index-*.json || echo "0 matches (good)"
   grep -c "dev-hub" build/search-index-*.json || echo "0 matches (good)"
   ```

6. Verify real content is still indexed:
   ```bash
   grep -c "Obtain Access Token" build/search-index-*.json
   grep -c "SIM card" build/search-index-*.json
   ```

7. Measure and record final index size (per D-08, D-09):
   ```bash
   ls -lh build/search-index-*.json
   # Also measure gzipped size (what CloudFront actually serves)
   gzip -c build/search-index-*.json | wc -c
   ```
   Record these values in the SUMMARY for Phase 14 optimization decisions.
  </action>
