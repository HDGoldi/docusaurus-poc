# Phase 13 — UI Design Contract

## Component Inventory

Phase 13 introduces exactly one new UI component via the search plugin. No custom components are built.

| Component | Source | Behavior |
|-----------|--------|----------|
| SearchBar | `@easyops-cn/docusaurus-search-local` (auto-injected) | Renders in navbar right section, next to "Terms and Abbreviations" link |

### Search Bar Placement

- **Location:** Main navbar, right side, inside the Docusaurus `SearchBar` component slot
- **Position:** After the last navbar item ("Terms and Abbreviations"), before the color mode toggle (if present)
- **Keyboard shortcut:** Cmd+K (macOS) / Ctrl+K (Windows/Linux) opens search modal
- **Source:** Plugin auto-injects into Docusaurus SearchBar slot. No modification to `src/theme/Navbar/Layout/index.tsx` needed.

### Search Modal (Plugin Default)

- **Trigger:** Click search bar or press Cmd/Ctrl+K
- **Overlay:** Dimmed backdrop behind modal (plugin default)
- **Input:** Text field with type-ahead, instant results on each keystroke
- **Results:** List of matching pages with context snippets showing surrounding text
- **Navigation:** Click result to navigate to page; both `/docs/` and `/api/` paths supported
- **Dismiss:** Click backdrop, press Escape, or click outside modal
- **Mobile:** Plugin handles responsive layout by default

## Interaction Contracts

### Search Flow

1. User sees search bar in navbar (always visible on all pages)
2. User clicks search bar OR presses Cmd/Ctrl+K
3. Modal overlay opens with text input focused
4. User types query text
5. Results appear below input as type-ahead (after each keystroke)
6. Results show page title + context snippet from both `/docs/` and `/api/` pages
7. User clicks a result OR navigates with arrow keys + Enter
8. Browser navigates to the selected page
9. Modal closes automatically on navigation

### States

| State | Behavior |
|-------|----------|
| Idle | Search bar visible in navbar with placeholder text |
| Active (empty input) | Modal open, input focused, no results shown |
| Active (typing) | Results appear as type-ahead below input |
| No results | Plugin default "No results" message displayed |
| Loading | Brief spinner while index loads on first search interaction (lazy-loaded) |
| Result selected | Highlighted row; Enter or click navigates |
