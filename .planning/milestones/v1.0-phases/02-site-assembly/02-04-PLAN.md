**A. Create `scripts/12-generate-redirect-map.ts`** (per D-05).

This script generates a JSON file mapping old ReadMe.com URL paths to new Docusaurus paths. The mapping is for future use with a CloudFront Function for URL redirects (Phase 3 or v2).

The script should:
1. Read the existing slug map (from `scripts/utils/slug-map.ts`) which maps ReadMe slugs to file paths
2. For each slug, compute:
   - **Old path**: The ReadMe.com URL pattern (typically `/docs/{slug}` for docs, `/reference/{slug}` for API)
   - **New path**: The Docusaurus URL based on the new multi-instance routing:
     - Docs in documentation instance: `/docs/{folder}/{slug}`
     - Docs in platform instance: `/platform/{folder}/{slug}`
     - Docs in blueprints instance: `/blueprints/{folder}/{slug}`
     - Docs in terms instance: `/terms/{slug}`
     - API docs: `/api/{spec}/{endpoint}`
3. Output the mapping as JSON to `static/redirect-map.json` in this format:
```json
{
  "redirects": [
    { "old": "/docs/device-controller-api", "new": "/platform/1nce-os/1nce-os-device-controller/device-controller-api" },
    { "old": "/docs/sim-cards-knowledge", "new": "/docs/sim-cards/sim-cards-knowledge" }
  ],
  "generated": "2026-03-20",
  "count": 298
}
```
4. Add npm script: `"convert:redirects": "tsx scripts/12-generate-redirect-map.ts"`
