## Spacing Scale

Docusaurus Infima manages its own spacing system via `--ifm-spacing-*` tokens. This phase does NOT override Infima's spacing defaults -- it uses them as-is. The following scale documents the effective values for reference by downstream consumers.

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, sidebar item padding |
| md | 16px | Default element spacing, card padding, paragraph margin |
| lg | 24px | Section padding, content area margins |
| xl | 32px | Major section breaks, content top margin |
| 2xl | 48px | Page-level vertical spacing (footer/header padding) |
| 3xl | 64px | Hero-level spacing (not used in this phase) |

Exceptions:
- Footer uses `1rem` (16px) vertical padding with `3rem` (48px) gap between items -- matches current site footer layout (source: `readme_custom.css` `.footer-content`)
- Sidebar hover/active items use 15px font with Infima default padding -- no custom spacing override needed

**Source:** Infima defaults. No custom spacing overrides in `readme_custom.css` that need porting to Docusaurus.

| Text (heading H2-H3) | `#1e4d7d` | H2 and H3 headings |
| Destructive | `#e3342f` (Infima default) | Not actively used in this phase |

### Dark Mode

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#1b1b1d` | Page background |
| Secondary (30%) | `#0d2a47` (darker navy) | Navbar background, footer background |
| Accent (10%) | `#29abe2` (1NCE light blue) | Same reserved list as light mode |
| Text (heading) | `#29abe2` | All headings stay teal per D-14 |

### Accent Reserved For

The accent color `#29abe2` is reserved exclusively for:
1. **Links** -- all inline text links and sidebar active links
2. **H1 headings** in light mode, **all headings** in dark mode
3. **Navbar hover state** -- tab hover color
4. **Footer link hover state**
5. **Primary button hover state** (if any CTA buttons exist in content)
6. **Docusaurus color-primary scale** -- the `--ifm-color-primary` family drives all Infima interactive elements (toggle, pagination, breadcrumb active)

Accent is NOT used for: body text, backgrounds (except sidebar hover at 20% opacity `#29abe232`), borders, or decorative elements.

## Interaction Contracts

| Interaction | Behavior |
|-------------|----------|
| Navbar tab click | Navigate to first page of that docs instance; active tab highlighted |
| Sidebar category click | Toggle expand/collapse; active page auto-expanded on load |
| Sidebar link click | Navigate to doc page; URL updates to clean path |
| Dark mode toggle | Instant theme switch via CSS custom properties; persists in localStorage |
| API Try It "Send" button | Execute HTTP request to API server URL from spec; display response inline |
| API auth field | Empty by default; developer enters Bearer token manually; persists within session |
| Footer link click | Opens external URL in new tab (`target="_blank"`) |
| Broken link (build) | `onBrokenLinks: 'throw'` -- build fails, no runtime 404s for internal links |
| Internal doc link click | Client-side navigation (SPA); triggers `onRouteDidUpdate` for analytics |
