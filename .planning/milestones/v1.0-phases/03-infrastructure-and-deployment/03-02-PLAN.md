**Job 1: `validate`** (runs on ALL pushes and PRs per D-03):
- `runs-on: ubuntu-latest`
- Steps:
  1. `actions/checkout@v4`
  2. `actions/setup-node@v4` with `node-version: 20` and `cache: npm`
  3. `run: npm ci`
  4. `run: npm run build` (Docusaurus build -- `onBrokenLinks: 'throw'` already handles broken link detection per D-05)
  5. `uses: actions/upload-artifact@v4` with `name: build-output`, `path: build/`, `retention-days: 1`
  6. `uses: treosh/lighthouse-ci-action@v12` with `uploadArtifacts: true` and `configPath: ./lighthouserc.json` (per D-05)

**Job 2: `deploy-preview`** (PRs only per D-02):
- `if: github.event_name == 'pull_request'`
- `needs: validate`
- `runs-on: ubuntu-latest`
- `environment: preview`
- Steps:
  1. `uses: actions/download-artifact@v4` with `name: build-output`, `path: build/`
  2. `uses: aws-actions/configure-aws-credentials@v6` with `role-to-assume: ${{ vars.AWS_ROLE_ARN }}`, `aws-region: us-east-1` (per D-04)
  3. `run: aws s3 sync build/ s3://1nce-developer-hub-preview/pr-${{ github.event.number }}/ --delete` (per D-15, D-16)
  4. Comment on PR with preview URL:
     ```yaml
     - name: Comment preview URL
       uses: actions/github-script@v7
       with:
         script: |
           github.rest.issues.createComment({
             issue_number: context.issue.number,
             owner: context.repo.owner,
             repo: context.repo.repo,
             body: `Preview deployed: https://${{ vars.PREVIEW_CF_DISTRIBUTION_DOMAIN }}/pr-${{ github.event.number }}/`
           })
     ```

**Job 3: `deploy-production`** (main branch only per D-01):
- `if: github.ref == 'refs/heads/main' && github.event_name == 'push'`
- `needs: validate`
- `runs-on: ubuntu-latest`
- `environment: production` (GitHub environment protection rule provides manual approval per D-01)
- Steps:
  1. `uses: actions/download-artifact@v4` with `name: build-output`, `path: build/`
  2. `uses: aws-actions/configure-aws-credentials@v6` with `role-to-assume: ${{ vars.AWS_ROLE_ARN }}`, `aws-region: us-east-1` (per D-04)
  3. Sync hashed assets with long cache:
     `aws s3 sync build/assets/ s3://1nce-developer-hub-prod/assets/ --cache-control "public, max-age=31536000, immutable"` (per D-08)
  4. Sync everything else with short cache:
     `aws s3 sync build/ s3://1nce-developer-hub-prod/ --exclude "assets/*" --cache-control "public, max-age=600, must-revalidate" --delete`
  5. Invalidate CloudFront cache:
     `aws cloudfront create-invalidation --distribution-id ${{ vars.CF_DISTRIBUTION_ID }} --paths "/*"` (per D-08, INFRA-07)
  6. Run smoke test:
     `bash scripts/smoke-test.sh https://${{ vars.PROD_CF_DISTRIBUTION_DOMAIN }}` (per D-12)

**SECOND FILE:** Create `scripts/smoke-test.sh` per D-12 -- a pre-launch script that verifies key URLs return 200.

```bash
#!/usr/bin/env bash
# Pre-launch smoke test: verify key URLs return 200 on target domain
# Usage: smoke-test.sh <base-url>
# Example: smoke-test.sh https://d1234567890.cloudfront.net
set -euo pipefail

BASE_URL="${1:?Usage: smoke-test.sh <base-url>}"

# Key URLs matching the 5 docs plugin instances + homepage
URLS=(
  "/"
  "/docs/1nce-os/1nce-os-overview/"
  "/api/"
  "/platform/"
  "/blueprints/"
  "/terms/"
)

FAILED=0
for path in "${URLS[@]}"; do
  STATUS=$(curl -s -o /dev/null -w "%{http_code}" --max-time 10 "${BASE_URL}${path}")
  if [ "$STATUS" -ne 200 ]; then
    echo "FAIL: ${BASE_URL}${path} returned $STATUS"
    FAILED=1
  else
    echo "OK: ${BASE_URL}${path}"
  fi
done

if [ "$FAILED" -eq 1 ]; then
  echo ""
  echo "SMOKE TEST FAILED: One or more URLs did not return 200"
  exit 1
fi

echo ""
echo "SMOKE TEST PASSED: All URLs returned 200"
exit 0
```
