## Content routing

Each docs section maps to a separate Docusaurus plugin instance with its own route prefix:

| Section       | Source path         | URL prefix     | Sidebar file              |
|---------------|---------------------|----------------|---------------------------|
| Documentation | `docs/documentation`| `/docs`        | `sidebars/documentation.ts` |
| API Explorer  | `docs/api`          | `/api`         | `sidebars/api.ts`         |
| Platform      | `docs/platform`     | `/platform`    | `sidebars/platform.ts`    |
| Blueprints    | `docs/blueprints`   | `/blueprints`  | `sidebars/blueprints.ts`  |
| Terms         | `docs/terms`        | `/terms`       | `sidebars/terms.ts`       |

## API docs generation

OpenAPI specs in `specs/*.json` are processed by `docusaurus-plugin-openapi-docs` and output into `docs/api/{spec-name}/`. The generated MDX files use `@theme/ApiItem` for rendering. Sidebar for API section uses `autogenerated` mode.

## Key conventions

- Documentation content is Markdown/MDX in `docs/`
- Each docs section has its own sidebar file in `sidebars/`
- The chat widget is mounted globally via `src/theme/Root.tsx`
- Static assets go in `static/img/`
- OpenAPI specs are the source of truth — edit `specs/*.json`, then regenerate API docs
- Infrastructure changes go through CloudFormation templates in `infra/`
